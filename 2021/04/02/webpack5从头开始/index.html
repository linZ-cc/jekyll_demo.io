webpack5从头开始

发表于 2021-04-02 更新于 2021-04-07

¶webpcak安装

  • 先使用npm init -y初始化文件
  • 然后安装本地的webpack
    npm install webpack webpack-cli -D

¶webpack可以进行0配置

  • 打包工具 -> 输出后的结果(js模块)
  • 使用npx webpack指令进行打包，默认调用mode_modules模块下的./bin/webpack.cmd,
    而webpack.cmd会运行node_modules/webpack/bin/webpack.js
  • 打包(支持我们的js的模块化)

¶手动配置webpack

  • 默认配置文件的名字 webpack.config.js
  • 运行时可手动指定配置文件 npx webpack --config webpack.config.my.js
  • 可以在package.json里面配置脚本

¶配置devServer (开发服务器配置)

  • 安装webpack-dev-server
    npm install webpack-dev-server -D
  • 在package.json中配置脚本
  • 在webpack.config.js中配置devServer
  • 运行npm run dev 或者 webpack serve --open 即可启动本地服务

¶添加html-webpack-plugin插件

  • 安装html-webpack-plugin,因为我们用的是webpack4，所以我们安装4.5.0的插件就好
    npm install html-webpack-plugin@4.5.0 -D
  • webpack.config.js中的配置

¶配置style-loader、css-loader模块

  • 安装loader
    npm install css-loader style-loader -D
  • 在模块里配置规则

¶打包抽离css文件

  • 安装插件mini-css-extract-plugin
    npm install mini-css-extract-plugin -D
  • 配置文件头部引入插件
    const miniCssExtractPlugin = require('mini-css-extract-plugin')
  • 将插件放入plugins里面，然后在loader中使用
  • 自动添加样式前缀
    npm install postcss-loader autoprefixer -D
  • 在webpack.config.js中配置module的规则
  • 然后在根目录创建postcss.config.js配置文件

  • 运行npm run build即可看到打包完的文件中带有各浏览器前缀

  • 这时，我们发现css文件没有压缩，所以接下来我们要压缩一下css文件，先安装optimize-css-assets-webpack-plugin插件
    npm install optimize-css-assets-webpack-plugin -D

  • 在webpack.config.js配置文件中新增optimization优化项, 需要在production模式下才生效

  • 打包完之后发现，css确实是压缩了，但是js却没有压缩，所以接下来我们需要安装uglifyjs-webpack-plugin插件
    npm install uglifyjs-webpack-plugin -D
  • 使用插件
  • 紧接着发现ES6语法好像没有转化，所以安装babel来转换一下
    npm isntall babel-loader @babel/core @babel/preset-env @babel/plugin-proposal-class-properties @babel/plugin-proposal-decorators -D
  • 在rules中配置一下, 如果要看转化效果，可以使用开发模式更为直观一点(没有压缩代码)
