¶DevTools failed to parse SourceMap

  • 使用webpack-dev-server开发时，浏览器控制台会报一个警告：DevTools failed to load SourceMap: Could not load content for webpack:///node_modules/sockjs-client/dist/sockjs.js.map: HTTP error: status code 404, net::ERR_UNKNOWN_URL_SCHEME

  • SourceMap技术
    SourceMap本质上是一个信息文件，里面存储这代码转换前后的对应位置，是源码和生产代码的映射。SourceMap解决了在打包过程中，代码经过压缩，去空格以及babel编译转化后，由于代码之间代码差异过大，造成无法debug的问题。SourceMap在代码处理前与处理后之间构建了一座桥梁，方便定位生产环境中出现的bug位置。

  • 解决方法：
    在webpcak.config.js中配置devtool: 'source-map即可

    ¶安装html-webpack-plugin配置报错 TypeError: Cannot read property 'tap' of undefined

    • 开始使用npm install html-webpack-plugin -D安装完插件后，在webpack.config.js中配置完plugins后运行报错
    • 在网上找了圈问题，发现是版本兼容问题,看了下安装的版本是^5.3.1的，而我们现在使用的webpack是^4.46.0的，所以降了下html-webpack-plugin的版本为^4.5.0解决
