webpack3-to-webpack4

发表于 2021-04-12 更新于 2021-04-14

¶先升级webpack3到webpack4

  • 将webpack3和webpack-dev-server卸载
    npm uninstall webpack webpack-dev-server
  • 安装webpack4和webpack-dev-server，webpack4需要安装脚手架
    npm install webpack@4.46.0 webpack-cli@4.6.0 webpack-dev-server@3.11.2 -D
    安装完之后需要将package.json文件中的script的dev指令修改一下
  • 在webpack.config.js中配置devServer, 如下所示：
  • webpack4新增mode属性配置，需要配置该属性，不然将报警告

¶运行npm run dev发现报错TypeError: compilation.mainTemplate.applyPluginsWaterfall is not a function

  • 原来html-webpack-plugin版本与webpack4不兼容，升级html-webpack-plugin
    npm uninstall html-webpack-plugin
    npm install html-webpack-plugin@4.5.0 -D

¶紧接着运行发现报错Cannot read property 'eslint' of undefined at Object.module.exports (C:\Users\ASUS\Desktop\web-admin\node_modules\eslint-loader\index.js:148:18)

  • 发现是eslint-loader与webpack4不兼容，安装新的eslint-loader
    npm uninstall eslint-loader
    npm install eslint-loader@2.1.2 -D

¶接着遇到了vue的相关的报错TypeError: Cannot read property 'vue' of undefined at Object.module.exports (C:\Users\ASUS\Desktop\web-admin\node_modules\vue-loader\lib\loader.js:57:18)

  • 查了下发现是webpack4需要升级vue-loader@13.5.0为vue-loader@14.2.2
    npm uninstall vue-loader
    npm install vue-loader@14.2.4 -D

¶遇到项目运行内存溢出Last few GCs JavaScript heap out of memory

  • 在终端执行set NODE_OPTIONS=--max_old_space_size=8172解决

¶又遇到了一些loader需要更新，现安装最新版本，分别为url-loader@4.1.1、file-loader@6.2.0、svg-sprite-loader@6.0.5

¶运行npm run dev运行成功，但是页面空白，查看控制台发现Module../node_modules/webpack-dev-server/client/clients/BaseClient.js

  • 搜了一圈后发现注释掉webapck配置文件中的rules的resolve('node_modules/webpack-dev-server/client')就可以

¶接着想生产环境打个包试试，发现报错webpack.optimize.CommonsChunkPlugin has been removed, please use config.optimization.splitChunks instead

  • webpack4已经移除了CommonsChunkPlugin,新增了optimization,于是先简单配置下optimization
    将原有配置CommonsChunkPlugin的配置项先注释,然后新增optimization配置

¶然后运行继续报错Error: Chunk.entrypoints: Use Chunks.groupsIterable and filter by instanceof Entrypoint instead

  • 此处好像是extract-text-webpack-plugin与webpack4版本不兼容，使用mini-css-extract-plugin
    npm install mini-css-extract-plugin -D
  • 在utils.js中引入并替换extract-text-webpack-plugin
  • 在webpack.prod.conf.js中配置

¶运行npm run build:prod后报错Unhandled rejection Error: "dependency" is not a valid chunk sort mode

  • 在html-webpack-plugin插件配置中将chunksSortMode: 'dependency'改为chunksSortMode: 'auto'

¶紧接着运行发现提示了另一个错误TypeError: compilation.contextDependencies.push is not a function,看上面的提示为.C:\Users\ASUS\Desktop\web-admin-webpack4\node_modules\copy-webpack-plugin\dist\index.js:151

  • 我们发现是copy-webpack-plugin的问题,将它升级一下 npm install copy-webpack-plugin@6.4.1 -D`
  • 在webpack.prod.conf.js中引入并使用

¶打包问题解决，回到运行dev试一下，发现菜单栏小logo没出现

  • 我以为是sass版本问题，于是把node-sass sass-loader都升级了下发现并没有解决
  • 于是我又把css-loader升级到^4.3.0，发现页面样式乱了…
  • 然后也没有提示什么错误，看了下vue-style-loader好像没有升级，于是升级到^4.1.3，前面的问题全部解决
  • 顺便把less less-loader更新一下
